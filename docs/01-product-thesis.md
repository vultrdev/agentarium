# Agentarium — Product Thesis

## Working Definition

Agentarium is a spatial operating system for AI agents.

It is not a game first.
It is not a metaverse first.
It is not a dashboard with 3D wallpaper.

It is a live world where agents can think, build, debate, inspect, and leave behind artifacts that matter.

## Core Promise

Users should be able to:
- watch agents operate in real time
- understand what each agent is doing and why
- intervene naturally through space, chat, or approvals
- move ideas through rooms that represent real work modes
- preserve outputs as reusable artifacts, tools, plans, and world objects
- replay important sessions as content, learning material, or audit trails

## Product Principles

### 1. Utility beats spectacle
If a feature is visually impressive but does not improve understanding, control, output quality, or replayability, it is decoration.

### 2. World state must reflect real state
Agent movement, room changes, lights, props, and UI overlays should be driven by runtime events, not fake ambient animation.

### 3. Artifacts matter more than chat
Chat is transient. Agentarium should produce durable outputs:
- plans
- specs
- notes
- code patches
- apps
- memories
- design assets
- recorded debates
- reusable workflows

### 4. Spatial semantics matter
Rooms are not cosmetics. Each room should map to a real operational mode.

### 5. Replay is a feature, not an afterthought
Every meaningful session should be reconstructable as:
- a timeline
- a playback
- a summary
- a shareable content moment

## Target Users

### Primary
- solo founders using AI as their team
- technical operators running many agents
- builders who want visibility into autonomous systems

### Secondary
- teams demoing agent workflows
- educators showing how agents reason and collaborate
- creators shipping regular AI-native content

## Why This Wins
Most agent products are hidden behind:
- chat logs
- terminal output
- dashboards
- isolated tools

Agentarium makes agent work legible through:
- space
- motion
- role identity
- event-driven world feedback
- persistent artifacts

## What It Is Not
- not a generic 3D office clone
- not a pure no-code builder
- not a toy NPC simulator
- not an animation layer disconnected from execution

## Initial Rooms
- **Forge** — building and output generation
- **Library** — retrieval, memory, research, synthesis
- **Arena** — debate, critique, agent-vs-agent review
- **Vault** — saved artifacts, templates, and approved outputs
- **Studio** — capture, replay, and content publishing

## Core Agent Archetypes
- **Architect** — decomposes goals, frames plans
- **Builder** — executes, writes, creates
- **Critic** — finds flaws, edge cases, bad assumptions
- **Researcher** — gathers evidence and context
- **Curator** — promotes outputs into durable artifacts
- **Janitor** — manages cleanup, resets, archival, stale state

## MVP Boundary
The MVP is:
- one world
- one main room
- three visible agents
- one event pipeline
- one artifact system
- one replayable session timeline

Anything beyond that is optional until the core loop is fun and useful.

## Success Criteria for MVP
A user can:
1. enter an idea
2. watch three agents collaborate around it
3. understand the process live
4. approve or redirect work
5. receive a durable artifact at the end
6. replay the session later

If that works, Agentarium is real.
If not, more rooms and prettier graphics will not save it.
