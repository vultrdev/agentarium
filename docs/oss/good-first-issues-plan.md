# Agentarium Good First Issues Plan

## Purpose

Identify the kinds of early issues that make the project welcoming to contributors.

## Good First Issue Characteristics

A good first issue should:
- touch one layer only
- have clear acceptance criteria
- avoid core contract churn
- be testable or visibly verifiable

## Good First Issue Categories

### Docs
- improve README examples
- clarify room semantics
- add glossary entries
- add sequence diagrams for runtime events

### UI / UX
- improve inspector panels
- improve task creation form
- add better approval prompt styling
- improve bookmark/replay visuals

### World polish
- better camera presets
- smoother semantic anchor transitions
- clearer artifact pedestal state visuals
- room focus highlighting

### Replay
- bookmark filters
- replay event grouping
- improved timeline labels

### Themes / assets
- new prop packs
- alternate Forge theme
- iconography improvements

## Things That Are Bad First Issues
- rewriting the world state model
- changing core event semantics
- replacing the engine stack
- adding a whole new backend protocol without guidance

## Maintainer Practice

The repo should always maintain:
- a small queue of labeled good-first-issues
- a few room/theme tasks
- a few replay/tasks UX tasks
- a few docs tasks
